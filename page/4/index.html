数据表的主键选择

发表于 2018-08-12 | 分类于 mysql | | 阅读次数:

关于数据表主键的设计，团队内部产生过这样的争论：

举例：订单表业务相关的字段有

字段 说明
amount 流水号
sn 流水号
pay_sn 支付流水号
buyer 购买人
ship 物流相关数据
create_at 订单生成时间

可以确定的是，sn 是唯一的，所以 A 的观点是 “用 sn 做为表的主键”，与其不同的观点是 “添加业务无关的 id 自增字段用作主键”。这两种设计哪个更合理呢？

简而言之，就是选择业务主键还是代理主键的问题。
